Eumel — Flowchart

Графические схемы игры. Открой этот файл в браузере двойным кликом или через npm run docs.

1. Общий поток игры

flowchart TD
    START([Старт игры]) --> SETUP[Создать игроков 4–6]
    SETUP --> PHASE_UP[Фаза 1: пирамида вверх
1 → max карт]
    PHASE_UP --> CHECK_UP{Ещё раунды
в фазе 1?}
    CHECK_UP -->|да| ROUND
    CHECK_UP -->|нет| PHASE_DOWN[Фаза 2: пирамида вниз
max−1 → 1]
    PHASE_DOWN --> CHECK_DOWN{Ещё раунды
в фазе 2?}
    CHECK_DOWN -->|да| ROUND
    CHECK_DOWN -->|нет| PHASE_ORDER[Фаза 3: заказы
каждый игрок выбирает N карт]
    PHASE_ORDER --> CHECK_ORDER{Все заказали
свой раунд?}
    CHECK_ORDER -->|нет| ROUND
    CHECK_ORDER -->|да| FINAL_BONUS[Финал: +10 всем
с макс. числом угадываний]
    FINAL_BONUS --> END([Eumel / победитель])
    ROUND --> CHECK_UP
    ROUND --> CHECK_DOWN
    ROUND --> CHECK_ORDER

2. Один раунд

flowchart TD
    R_START([Начало раунда]) --> DEAL[Тасовка + раздача N карт
остаток остаётся в колоде]
    DEAL --> VIS{Карт
на игрока?}
    VIS -->|1| VIS_ONE[Все видят чужие карты
свою — нет]
    VIS -->|2+| VIS_HAND[Каждый видит только свою руку]
    VIS_ONE --> BID
    VIS_HAND --> BID[Ставки по очереди]
    BID --> BID_FIRST{Первый раунд
игры?}
    BID_FIRST -->|да| BID_P1[Первым ставит игрок 1]
    BID_FIRST -->|нет| BID_LOSER[Первым ставит проигравший
прошлого раунда]
    BID_P1 --> BID_LOOP
    BID_LOSER --> BID_LOOP[Игроки ставят по кругу]
    BID_LOOP --> BID_LAST{Последний
ставящий?}
    BID_LAST -->|нет| BID_LOOP
    BID_LAST -->|да| BID_CHECK{Сумма ставок
допустима?}
    BID_CHECK -->|нет| BID_REJECT[Отклонить ставку
выбрать другую]
    BID_REJECT --> BID_CHECK
    BID_CHECK -->|да| TRICKS
    TRICKS[Взятки: первым ходит
последний ставивший] --> TRICK_LOOP{Ещё взятки
в раунде?}
    TRICK_LOOP -->|да| PLAY[Игроки ходят по кругу
следование масти обязательно]
    PLAY --> WIN[Определить победителя взятки
♠ > ♣ > ♦ > ♥, старший ранг]
    WIN --> TRICK_LOOP
    TRICK_LOOP -->|нет| SCORE[Подсчёт очков раунда]
    SCORE --> SCORE_OK{Угадал
ставку?}
    SCORE_OK -->|да| SCORE_BONUS[score += 10 + tricksWon
correctBids++]
    SCORE_OK -->|нет| SCORE_PLAIN[score += tricksWon]
    SCORE_BONUS --> ROUND_END
    SCORE_PLAIN --> ROUND_END[Проигравший раунда →
первый в ставках следующего]
    ROUND_END --> R_END([Конец раунда])

3. Проверка суммы ставок

flowchart TD
    INPUT[Последний игрок выбирает ставку] --> SUM[Сумма всех ставок]
    SUM --> CENTRAL{Центральный
раунд?
5–6 max-карт}
    CENTRAL -->|нет| RULE1{sum === tricks?}
    RULE1 -->|да| REJECT[Недопустимо]
    RULE1 -->|нет| OK[Ставка принята]
    CENTRAL -->|да| RULE2{sum === tricks
или tricks±1?}
    RULE2 -->|да| REJECT
    RULE2 -->|нет| OK

4. Подсчёт взятки

flowchart TD
    LEAD[Первая карта задаёт масть хода] --> NEXT{Ещё карты
в взятке?}
    NEXT -->|да| FOLLOW{У игрока есть
масть хода?}
    FOLLOW -->|да| MUST[Обязан сыграть масть хода]
    FOLLOW -->|нет| ANY[Может сыграть любую]
    MUST --> COMPARE
    ANY --> COMPARE[Сравнить с текущим лидером
card.beats]
    COMPARE --> NEXT
    NEXT -->|нет| WINNER[Победитель забирает взятку
tricksWon++]

5. Финал игры

flowchart TD
    FIN([Все раунды сыграны]) --> MAX[Найти max correctBids]
    MAX --> BONUS[+10 score всем
с correctBids === max]
    BONUS --> MIN[Найти min score]
    MIN --> EUMEL[Eumel: все с score === min]
    MIN --> WIN[Победитель: все с score === max]
